# Domain Model

## Core Entities

### Organization
The tenant. A startup using AgentCFO.

```
Organization
├── Id (Guid)
├── Name (string)
├── StripeCustomerId (string)
├── StripeAccountId (string)        # For Connect scenarios
├── MonthlyBudget (Money)
├── RunwayThresholdDays (int)       # Alert when runway < this
├── CreatedAt (DateTime)
└── Policies[]                      # Agent governance rules
```

### Transaction
A financial event — money in or money out.

```
Transaction
├── Id (Guid)
├── OrganizationId (Guid)
├── StripeEventId (string)          # Idempotency key from Stripe
├── Type (enum: Revenue | Expense | Refund | Fee)
├── Amount (Money)
├── Currency (string)
├── Description (string)
├── Category (enum: Subscription | OneTime | Infrastructure | Marketing | 
│              Tools | Contractors | Office | Legal | Other)
├── Status (enum: Pending | Completed | Failed | Refunded)
├── StripePaymentIntentId (string?)
├── StripeInvoiceId (string?)
├── OccurredAt (DateTime)
└── Metadata (Dictionary)           # Stripe metadata passthrough
```

### Budget
Spending limits per category, per period.

```
Budget
├── Id (Guid)
├── OrganizationId (Guid)
├── Category (ExpenseCategory)
├── MonthlyLimit (Money)
├── CurrentSpend (Money)            # Computed from transactions
├── Period (enum: Monthly | Quarterly)
├── AlertThresholdPercent (int)     # Warn at 80% etc.
├── HardLimit (bool)                # If true, agent cannot exceed
└── IsActive (bool)
```

### Forecast
Cash flow projection — generated by the agent.

```
Forecast
├── Id (Guid)
├── OrganizationId (Guid)
├── GeneratedAt (DateTime)
├── GeneratedBy (enum: Agent | Manual)
├── CurrentCashBalance (Money)
├── MonthlyBurnRate (Money)
├── MonthlyRevenue (Money)
├── RunwayDays (int)
├── RunwayEndDate (DateOnly)
├── Scenario (string?)              # "base", "+2 engineers", etc.
├── ProjectionPoints[]              # 30/60/90 day data points
│   ├── Date (DateOnly)
│   ├── ProjectedBalance (Money)
│   ├── ProjectedRevenue (Money)
│   └── ProjectedExpenses (Money)
└── Confidence (enum: Low | Medium | High)
```

### AgentDecision
An action the agent took, with full reasoning.

```
AgentDecision
├── Id (Guid)
├── OrganizationId (Guid)
├── Type (enum: ExpenseApproved | ExpenseDenied | BudgetAdjusted |
│         AnomalyDetected | ReportGenerated | InvoiceCreated |
│         PaymentRetryInitiated | ForecastUpdated | AlertRaised)
├── Description (string)            # Human-readable summary
├── Reasoning (string)              # Agent's explanation
├── RelatedTransactionId (Guid?)
├── RelatedBudgetId (Guid?)
├── Status (enum: Proposed | Executed | Overridden | Failed)
├── ProposedAt (DateTime)
├── ExecutedAt (DateTime?)
├── OverriddenBy (string?)          # Human who overrode
└── Metadata (Dictionary)
```

### AuditEntry
Immutable log of everything that happens.

```
AuditEntry
├── Id (Guid)
├── OrganizationId (Guid)
├── ActorType (enum: Agent | Human | System | Webhook)
├── ActorId (string)                # Agent ID, user ID, etc.
├── Action (string)                 # e.g., "transaction.categorized"
├── EntityType (string)             # e.g., "Transaction"
├── EntityId (Guid)
├── BeforeState (JSON?)             # Snapshot before change
├── AfterState (JSON?)              # Snapshot after change
├── Timestamp (DateTime)
└── CorrelationId (Guid)            # Links related events
```

## Value Objects

```
Money
├── Amount (decimal)
├── Currency (string)
└── Methods: Add(), Subtract(), Convert()

DateRange
├── Start (DateOnly)
├── End (DateOnly)
└── Methods: Contains(), Overlaps(), Days()

MoneyThreshold
├── Amount (Money)
├── Direction (enum: Above | Below)
└── Methods: IsBreached(Money)
```

## Domain Events

```csharp
// Raised when Stripe webhook arrives
public record PaymentReceived(Guid TransactionId, Money Amount, string StripePaymentIntentId);

// Raised when agent categorizes a transaction
public record TransactionCategorized(Guid TransactionId, ExpenseCategory Category, string Reasoning);

// Raised when spending approaches or exceeds budget
public record BudgetThresholdBreached(Guid BudgetId, Money CurrentSpend, Money Limit, int PercentUsed);

// Raised when runway drops below threshold
public record RunwayAlert(int RunwayDays, DateOnly ProjectedEndDate, string Severity);

// Raised when agent makes a governance decision
public record AgentDecisionMade(Guid DecisionId, AgentDecisionType Type, string Reasoning);

// Raised when Stripe payment fails
public record PaymentFailed(string StripePaymentIntentId, string FailureReason, int RetryCount);
```

## Key Relationships

```
Organization 1──* Transaction
Organization 1──* Budget
Organization 1──* Forecast
Organization 1──* AgentDecision
Organization 1──* AuditEntry
Transaction  1──0..1 AgentDecision    # Decision may relate to a transaction
Budget       1──* Transaction         # Transactions consume budget
Forecast     *──1 Organization
```

## Notes

- **Money as value object**: Never use raw decimal for currency. Always pair with currency code.
- **Audit is mandatory**: Every state change produces an AuditEntry. MediatR pipeline behavior handles this automatically.
- **Agent decisions are proposals first**: Agent proposes → policy engine validates → then executes. This is the core safety model.
- **Stripe events are idempotent**: Use StripeEventId to prevent duplicate processing.
